# BurrowScope App Instructions

## Purpose

This app runs the exported Bank Swallow burrow YOLO segmentation model in the browser with ONNX Runtime Web.

It is for per-image burrow segmentation and visual review. It does not solve duplicate counting across overlapping photos.

## Main Folder

App folder:

`D:\Bank_web1`

Production ONNX model used by the app:

`D:\Bank_web1\models\bank_swallow_burrow_yolo11s_seg_production.onnx`

Original exported model:

`D:\IA_videos\modelos\bank_swallow_burrow_yolo11s_seg_production.onnx`

## Run Locally

Open PowerShell and run:

```powershell
cd D:\Bank_web1
python -m http.server 8080
```

Then open:

`http://localhost:8080`

Do not open `index.html` directly by double click. Browser ONNX loading is more reliable through a local HTTP server.

## Basic Use

1. Open the local app.
2. Wait until the model panel says `ONNX ready`.
3. Upload one bank-wall image.
4. Adjust confidence if needed.
5. Run segmentation.
6. Review predicted masks visually.
7. Export CSV, JSON, or PNG if useful.

## Outputs From The App

The app can export:

- CSV with burrow detections, confidence, boxes, and centroids.
- JSON with the same detection information.
- PNG with the visible mask overlay.

These exports are generated by the browser when you click the export buttons.

## Scope

This app solves only:

`single image -> burrow masks, boxes, centroids, confidence values`

It does not yet solve:

`overlapping images -> duplicate merging -> unique burrow count`

The later counting pipeline should be:

`YOLO-seg predictions -> mask centroids and boxes -> image registration between overlapping photos -> shared coordinate system -> duplicate merging -> unique burrow count`

## Deployment Notes

For a simple internal app, any static server is enough.

The required files are:

- `index.html`
- `css/`
- `js/`
- `models/`
- `img/`
- `site.webmanifest`
- icons

The ONNX model is about 40 MB, so the web host must allow files of that size.

## Technical Note

The YOLO segmentation decoder is in:

`D:\Bank_web1\js\yolo-seg-postprocess.js`

If predicted masks look shifted or scaled wrong in a future browser test, this is the file to tune first.
